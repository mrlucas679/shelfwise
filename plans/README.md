## Audit verdict

The core application should remain alive. The receipt-driven domain engine, tool grounding,
HITL invariants, generated-world coverage, and MI300X dual-tier serving are substantive assets.
The release is not production-approved because CI is currently red, the public topology permits
insecure cookie transport by default, the GPU bootstrap does not enforce its documented source
allowlist, and the 15-minute harness runs in-process with memory stores rather than through the
deployed Postgres/Redis/Nginx topology.

## Deferred findings

- AMD VRAM telemetry is recorded around `148 MB` while the observed engines occupied roughly
  `148 GB`; normalize nested AMD-SMI units before using benchmark memory comparisons.
- `frontend/src/App.tsx` is 3,429 lines and has no browser E2E suite. Add Playwright after the
  deployed-topology gate exists, starting with auth, chat persistence, approvals, outage UI, and
  tenant isolation.
- README and HANDOFF evidence counts and model descriptions conflict with the final dual-tier
  runbook. Plan 001 updates the release-facing claims; a later docs pass should archive stale
  historical sections instead of continually appending new truth above old truth.
- Strong-tier JSON behavior is selected by the exact model string
  `google/gemma-4-31B-it`. Replace model-name sniffing with an explicit endpoint capability after
  the release blockers are closed.

## Findings considered and rejected

- Add more agents: rejected. The existing role split is already broad, and more roles would worsen
  context and tool-definition pressure without evidence of a missing decision domain.
- Replace Postgres/Redis with a graph database now: rejected. Recursive Postgres facts and the
  current event bus are adequate for this release; migrate only after measured query pressure.
- Run full 12B fine-tuning on the serving MI300X: rejected. Training remains a W7900 Jupyter concern;
  serving and pressure proof should not contend with training for one GPU.
- Dependency upgrade campaign: rejected for now. Frontend production `npm audit` reported zero
  advisories and `pip check` found no broken requirements; Python advisory scanning remains an
  explicit audit gap because `pip-audit` is not installed.

## Audit scope

Audited: backend/inference/harness hot paths, production Compose and Nginx, MI300X bootstrap,
benchmark telemetry, recovery tooling, frontend build/test posture, CI, capability contract,
README/HANDOFF/runbook drift, recent churn, and final run artifacts.
